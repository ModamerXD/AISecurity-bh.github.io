Automated LLM Testing Tool

ITCY 499 Senior Project: A dynamic fuzzing framework designed to extract hidden system prompts and test the safety guardrails of modern Large Language Models.

The Problem

As Artificial Intelligence becomes deeply integrated into enterprise systems, the hidden instructions (system prompts) guiding these models become critical intellectual property and security boundaries. Standard safety filters evaluate responses for general harm, but often fail to detect sophisticated data exfiltration techniques, leaving proprietary AI configurations vulnerable to extraction.

Core Capabilities

Dynamic Attack Generation

Utilizes algorithmic mutations (like many-shot, PAIR, and custom payloads) to automatically generate and refine adversarial prompts, bypassing static AI defenses.

System Prompt Extraction

A custom-engineered attack module specifically designed to trick target models (e.g., Llama 3.1) into dumping their initial context windows and hidden developer instructions.

Automated Evaluation

Features a specialized classifier system that accurately detects successful data leaks, identifying false negatives that standard semantic harm classifiers miss.

Example Execution

Our framework executes batch tests against local and cloud-based models. Below is an example of a prompt extraction attack targeting a local Llama-3.1 instance:

> python -m fuzzyai.cli fuzz -m ollama/llama3.1 -a prompt_extraction -T payloads.txt
